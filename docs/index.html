claude × codex, together.

duo is a tiny wrapper that runs Claude Code and Codex CLI as peers in one terminal — with multi-agent swarms, continuous cross-checking, and automatic credit fallback. One prompt, two agents, a swarm of sub-agents, a single shared transcript.

★ GitHub Install How it works

Why duo

peer to peer

Claude plans & validates. Codex executes. Both see the same transcript.

swarm by default

Each peer may spawn sub-agents in parallel for independent sub-tasks.

continuous check

Every delegated step is validated before the next — not fixed turns.

credit fallback

If one peer hits a rate-limit or quota, the other takes over cleanly.

Install & run

# Requires Python 3.10+ and both CLIs installed & logged in:
#   claude   (Claude Code)
#   codex    (Codex CLI)

pip install duox

duox "scaffold a python todo cli in ./scratch with pytest tests"
duox --once "fix the failing auth tests"        # one-shot, then exit
duox -p "summarise recent changes" --output-format json
duox doctor                                     # verify peers / MCP / readline

Safety

duo runs both CLIs with their most permissive flags so they can actually do work without interrupting you. Use it in a directory you trust — ideally a scratch repo, container, or VM. Read the transcript; these are still agents.
